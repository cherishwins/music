## THE FREE VALUE HOOK PROBLEM

### Current State: We Don't Have One

Right now, someone opens the app and... what? We need to answer:

> "What value do they get BEFORE we ask for money?"

### Option 1: Free Generation, Watermarked Download

```
PROS:
- They hear the full quality
- Can use watermarked version for testing
- Watermark annoys them into paying

CONS:
- Some will just use watermarked version
- Watermark might hurt brand perception
- Free usage = our costs ($0.23/track)

VERDICT: Viable, but must limit to 3/month
```

### Option 2: Free Preview, Paid Full Track

```
PROS:
- Preview is cheap/free to generate
- Only pay for what they'll actually use
- Clear value exchange

CONS:
- Preview might not show full quality
- Extra step in funnel
- "Just let me hear it first"

VERDICT: More complex, may reduce conversions
```

### Option 3: Freemium with Hard Limits

```
PROS:
- First track free, no watermark
- Experience full value once
- Clear upgrade path

CONS:
- Everyone takes the free track and leaves
- Free track might be enough for one launch
- High cost of free tier

VERDICT: Risky, but could work with good retention
```

## CONVERSION FUNNEL TESTING

### Experiment 1: First Visit Experience

**Hypothesis**: Removing ALL friction from first generation increases conversion

**Test A (Current)**:
- Open app → See welcome screen → Click generate → Enter prompt → Wait

**Test B (Zero Friction)**:
- Open app → Already on generation screen → One field visible → Generate button

**Metric**: Time to first generation, completion rate

### Experiment 2: Watermark Aggressiveness

**Hypothesis**: More annoying watermark = more conversions

**Test A**: Subtle watermark (3 second intro tag)
**Test B**: Aggressive watermark (intro + every 30 seconds)
**Test C**: Visual only (audio clean, but branded waveform)

**Metric**: Conversion rate, user complaints

### Experiment 3: Pricing Presentation

**Hypothesis**: Per-track pricing converts better than subscription-first

**Test A**: Show subscription first, per-track as fallback
**Test B**: Show per-track first, subscription as "save money"
**Test C**: Show both equally

**Metric**: Revenue per user, conversion rate

### Experiment 4: Payment Method Order

**Hypothesis**: Crypto-first users convert higher on TON

**Test A**: TON → Stars → Card (crypto-first)
**Test B**: Stars → TON → Card (familiar-first)
**Test C**: Detect wallet, show best option first

**Metric**: Conversion rate, revenue per conversion

## WEEKLY TESTING CADENCE

### Week 1: Baseline

- Launch with Test A (current experience)
- Track all funnel metrics
- Identify biggest drop-off point
- No changes, just measure

### Week 2: First Optimization

- Implement change to biggest drop-off
- A/B test with 50% of users
- Measure improvement
- Document learnings

### Week 3: Second Optimization

- Keep winner from Week 2
- Test next biggest drop-off
- Iterate

### Week 4: Scale

- Implement all winners
- Start paid acquisition (Telegram Ads)
- Measure CAC vs LTV

## THE HONEST TRUTH

We won't get this right on the first try. The plan is:

1. **Launch with best guess** (this doc)
2. **Measure everything** (funnel metrics)
3. **Talk to users** (why did you pay? why didn't you?)
4. **Iterate fast** (weekly improvements)
5. **Double down on what works** (scale winners)

The meme coin market is MASSIVE (13M+ tokens in 2025). If we capture just 0.01% of creators, that's 1,300 paying users.

The question isn't "is there demand?" - it's "can we convert that demand?"

This playbook is our hypothesis. Reality is the test.
